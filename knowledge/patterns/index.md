# Pattern Library Index

Cross-domain recurring patterns. Each entry is a pattern that has been observed in at least 2 distinct situations and has a non-obvious solution worth preserving.

---

## How to Use This Index

- **Finding a pattern**: scan the Domain column or use `/recall [pattern description]`
- **Adding a pattern**: use `/pattern [description]` to run the full capture protocol
- **Pattern IDs**: format `PAT-YYYY-NNN` — year of first observation, sequential within year

## Pattern Themes

As the library grows, patterns that cluster will be synthesized into cluster notes. Cluster threshold: 5+ patterns on a shared theme.

**Anticipated themes** (based on workspace domains):
- Decision patterns under uncertainty
- Context management and information routing
- Knowledge compounding and decay
- Stakeholder alignment without authority
- AI workflow reliability

## Anti-Patterns

Anti-patterns live in `architecture/ANTI-PATTERNS.md`, not here. The pattern library is for solutions. Anti-patterns are for named failure modes to avoid.
