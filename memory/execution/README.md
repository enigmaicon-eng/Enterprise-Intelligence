# memory/execution/

Task-type execution lessons. One file per task type, updated in place.

**What goes here:** What works and what fails when executing a specific type of task with a specific skill. Gotchas, input quality requirements, time estimates, pattern variations.

**What does NOT go here:** Per-workflow notes (use P15 timeline/journal), raw process notes, skill documentation (that lives in .claude/commands/).

**When to create:** After completing a workflow where something non-obvious was learned about how to do this type of work.

**Skill:** Write manually after workflow completion; `/mem-recall --type execution` to retrieve
**Schema:** `architecture/MEMORY-SCHEMAS.md` → Schema 4
**File naming:** `exec_TASKSLUG.md` (one per task type, never per workflow instance)
**Review interval:** 30 days from `last_updated`
**Update protocol:** Edit in place — do not create new file for same task type
**Archive trigger:** Task type is obsolete or skill has been deprecated
