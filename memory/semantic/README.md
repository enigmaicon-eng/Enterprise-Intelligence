# memory/semantic/

Working concept records. Current best understanding — mutable and subject to revision.

**What goes here:** Concepts that recur across multiple sessions where misunderstanding them has real cost. Current state of understanding, not final synthesis.

**What does NOT go here:** Stable domain knowledge (promote to knowledge/<domain>/ when stable), one-time facts, raw research notes.

**When to create:** When you find yourself re-deriving the same concept across sessions.

**Skill:** `/mem-recall` to retrieve; write manually or via `/mem-capture --type semantic`
**Schema:** `architecture/MEMORY-SCHEMAS.md` → Schema 2
**File naming:** `sem_TOPICSLUG.md` (one file per concept, updated in place)
**Review interval:** 30 days from `last_updated`
**Promotion trigger:** confidence=high and stable for 60+ days → `knowledge/<domain>/`
**Archive trigger:** confidence=low and not updated in 60 days
