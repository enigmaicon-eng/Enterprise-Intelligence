# Knowledge Compounding + Retrieval Intelligence System
## Making Knowledge Grow in Value Over Time

---

## What This System Is

P17 implements the compounding mechanism that turns a static knowledge base into a living intelligence system. The existing `knowledge/` corpus has 18+ high-quality PM entries, a cluster directory, and domain organization — but no connection layer, no retrieval scoring, and no mechanism for knowledge to build on itself.

P17 adds three things:
1. **Connection graph** — explicit, machine-readable relationships between entries
2. **Retrieval intelligence** — ranked, graph-aware search that traverses connections
3. **Compounding mechanisms** — knowledge that accumulates evidence, forms clusters, and synthesizes upward

---

## The Compounding Model

```
Single entry (standalone)
    ↓  /knowledge-connect (add connections)
Entry with neighbors (networked)
    ↓  threshold: ≥5 connected entries
Cluster candidate
    ↓  /knowledge-cluster (synthesize)
Cluster synthesis (higher-order entry)
    ↓  /insight, /pattern (cross-domain)
Strategic insight (P16 memory/insights/)
```

Value compounds at each level. A connected entry is worth more than an isolated one. A synthesized cluster captures insight that no single entry can express.

## Knowledge Graph Layer

**File:** `knowledge/KNOWLEDGE-GRAPH.json`
**Generated by:** `scripts/knowledge_index.py`

The graph stores what the markdown files cannot: relationships between entries, retrieval frequency, compound event counts, and cluster memberships.

### Node Schema
```json
{
  "id": "pm/product-strategy",
  "title": "Product Strategy",
  "domain": "pm",
  "file": "knowledge/pm/product-strategy.md",
  "confidence": "high",
  "reviewed": "YYYY-MM-DD",
  "last_accessed": "YYYY-MM-DD",
  "access_count": 0,
  "compound_events": 0,
  "cluster": [],
  "tags": []
}
```

### Edge Schema
```json
{
  "from": "pm/product-strategy",
  "to": "pm/metrics-experimentation",
  "type": "related_to | builds_on | contradicts | applies_to | synthesizes",
  "weight": 0.8,
  "created_at": "YYYY-MM-DD",
  "note": "Optional one-line note explaining the connection"
}
```

**Edge types:**
| Type | Meaning |
|------|---------|
| `related_to` | Covers overlapping concepts |
| `builds_on` | This entry requires the other as a prerequisite |
| `contradicts` | These entries contain conflicting evidence |
| `applies_to` | One entry is a context for applying the other |
| `synthesizes` | A cluster entry that synthesizes the target entry |

### Cluster Schema
```json
{
  "id": "cluster-pm-strategy-core",
  "name": "PM Strategy Core",
  "members": ["pm/product-strategy", "pm/metrics-experimentation"],
  "synthesis_file": "knowledge/clusters/pm-strategy-core.md",
  "last_synthesized": "YYYY-MM-DD",
  "synthesis_quality": "high | medium | low | null"
}
```

## Retrieval Intelligence

**Full algorithm:** `architecture/RETRIEVAL-INTELLIGENCE.md`

**Scoring formula:**

```
score = keyword_match(0–1) × 0.45
      + confidence_weight(0–1) × 0.20
      + recency_weight(0–1) × 0.15
      + connection_density(0–0.2) × 0.10
      + tag_match(0–1) × 0.10
```

**Connection density bonus:** +0.04 per direct neighbor, capped at 0.20.
**Recency weight:** 1.0 for reviewed within 30 days, decaying to 0.3 at 365+ days.
**Confidence weight:** high=1.0, medium=0.7, low=0.3.

After finding the primary match, traverse 1-hop neighbors and surface the top 3 as "Also see."

## Compounding Mechanisms

### 1. Connection Creation (`/knowledge-connect`)
Creates an edge between two entries. Logs to COMPOUNDING-LOG.md. Increments `compound_events` on both nodes. When a node reaches 5+ neighbors: suggests cluster formation.

### 2. New Learning Integration (`/learn` + connection surface)
When `/learn` creates a new knowledge entry, the system searches for related existing entries and surfaces the top 3 as connection candidates. The operator confirms which connections to add. This is the primary growth mechanism — new knowledge doesn't arrive in isolation.

### 3. Cluster Synthesis (`/knowledge-cluster`)
When a cluster of ≥5 connected entries is detected, `/knowledge-cluster` synthesizes them into a higher-order entry in `knowledge/clusters/`. The synthesis captures: shared themes, cross-entry patterns, tensions between entries, and the insight that no single entry expresses.

### 4. Spaced Review (`/knowledge-review`)
Surfaces entries due for review based on: time since `reviewed`, confidence level, and connection density. High-connection entries are reviewed more often (they underpin more of the graph). Review updates the `reviewed` field and can upgrade confidence.

### 5. Cross-Domain Bridge Detection (`/knowledge-connect --discover`)
Scans for entries in different domains that share tags or concepts without explicit connections. Surfaces these as "bridge candidates" — potential connections across domain boundaries that are the most valuable for compounding.

## Compounding Log

**File:** `knowledge/COMPOUNDING-LOG.md`
Append-only record of every compound event:
- Connection created (from → to, type, date)
- Cluster synthesis completed (cluster ID, member count, date)
- Entry enriched (entry ID, what was added)
- Review completed (entry ID, confidence change if any)

This log is the audit trail of knowledge growing.

## Integration Map

| Existing System | P17 Adds |
|----------------|---------|
| `knowledge/KNOWLEDGE-INDEX.md` | Unchanged; graph is the overlay, not the replacement |
| `knowledge/clusters/` | Clusters now machine-readable in graph; /knowledge-cluster populates it |
| `/recall` skill | Graph-aware upgrade: traverses connections, ranks by score |
| `/learn` skill | After writing a new entry: search for related entries, surface connection candidates |
| `/pattern` skill | Patterns link to knowledge entries via graph edges |
| P16 `memory/insights/` | Top cluster syntheses are promoted to insights via /insight or /pattern |

## Anti-Patterns

| Pattern | Why Forbidden |
|---------|--------------|
| Connecting every entry to every other | High-connectivity graphs lose signal; only create edges for genuine relationships |
| Synthesizing clusters before ≥5 members | Premature synthesis produces shallow entries; wait for sufficient evidence |
| Updating graph without knowledge_index.py | Manual graph edits drift from file reality; always regenerate |
| Treating graph as source of truth for entry content | Entries are canonical; graph is the overlay |
| Creating connections between low-confidence entries | Connects uncertainty to uncertainty; prioritize high-confidence connections |
