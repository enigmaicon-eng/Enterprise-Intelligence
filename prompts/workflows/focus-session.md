# Focus Session Prompts
## Version: v1.0

---

## Session Frame

```
I'm about to do a deep work session on: [problem or deliverable]

Frame it for me:

1. PROBLEM STATEMENT: Restate in one specific sentence. If vague, push me to sharpen it.
   Bad: "Work on the roadmap" 
   Good: "Write the H2 narrative section of the Q3 roadmap — specifically the rationale for de-prioritizing feature X"

2. DONE CONDITION: Observable output when the session ends.
   Must be completable in 60-90 minutes. If not: scope it down. What's the minimum valuable version?

3. CONTEXT TO LOAD: 1-3 files or items. No more.

4. SESSION RISK: One thing that could derail this session's focus. Name it.

5. START HERE: First specific action. Not "begin writing" — the actual first move.

Format:
FOCUS SESSION — [date]
Problem: [specific sentence]
Done condition: [observable output]
Context: [1-3 items]
Risk: [one thing]
Start here: [first action]
```

---

## Done Condition Sharpener

```
My done condition is: [what I wrote]

Is this an output (something that exists at the end) or a process (something I do)?

If it's a process → reframe as output. What will exist that doesn't exist now?

If it's an output:
  Is it completable in 60-90 minutes? [Yes | Probably not]
  If not: what's the minimum valuable version that IS completable in 60-90 minutes?

Give me: one sharper done condition.
```

---

## Session Close (post-session)

```
My session just ended.

Done condition was: [original done condition]

Quick close:
1. Done condition met? [Yes | Partial | No]
2. Current state (one sentence): [where things stand right now]
3. Next action (one sentence): [what the next session starts with]
4. Anything to route? [decisions / captures / knowledge candidates that surfaced]

If partial or no: what specifically remains? Should this roll to a new session today or tomorrow?
```

---

## Scope Reduction

```
I need to do deep work on [task] but I only have [time available].

The full task requires: [what you'd normally do]

Given [time] constraint:
1. What's the minimum valuable version of this task?
2. What must be done in this session vs. what can wait?
3. What's the specific done condition for the [time] version?

Output: scoped version with done condition + explicit list of what was deferred.
```

---

## Context Switching Record

```
I'm switching contexts from [task A] to [task B].

Before I switch — record where I am on [task A]:
- Current state: [where things are right now]
- Next action: [what I'd do next if I came back]
- Any open loops: [things I said I'd do but haven't yet]

File this as a context note. I'll use it as my re-entry point when I return to [task A].
```
