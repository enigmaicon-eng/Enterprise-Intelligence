# Daily Planning Prompts
## Version: v1.0

---

## Morning Plan

```
Read: execution/active-initiatives.md (current statuses only)
Read: notes/raw/ (any EOD capture from yesterday for the "tomorrow starts with" seed)

Generate my daily plan:

1. THEME: One phrase that orients today's cognitive focus.
   Not a task. A frame. Examples: "Stakeholder alignment," "Architecture clarity," "Execution push."

2. TOP 3 COMMITMENTS: Three specific outputs. Action verb + deliverable.
   Apply: commitment first, then leverage, then reversibility.
   Do not list more than 3. If I give you more than 3, apply the hierarchy and trim.

3. ONE RISK: The single most plausible thing that makes the top 3 impossible today.

4. CONTEXT TO LOAD: 1-3 files I'll need. No more.

Format:
PLAN — [date]
Theme: [phrase]

Commitments:
1. [output] → [commitment | leverage | reversibility]
2. [output] → [why]
3. [output] → [why]

Risk: [specific blocker]
Context to load: [files or topics]
```

---

## Weekly Plan (Monday)

```
Read: execution/active-initiatives.md
Read: strategy/active-bets.md (check for review-due bets this week)
Read: reviews/weekly/ — most recent exec review

Generate my weekly plan:

1. WEEK THEME: One phrase. What is this week oriented around?

2. TOP 3 WEEK COMMITMENTS: Week-level outputs — things that, if done this week, represent real progress.

3. STRATEGIC DEPENDENCY: One external dependency that could affect this week.
   What, from whom, by when.

4. DECISION THRESHOLD: One decision that, if delayed, costs more than deciding it sub-optimally.
   Name it. This decision should happen this week regardless of perfect information.

Format:
WEEK PLAN — [Week of YYYY-MM-DD]
Theme: [phrase]

Week commitments:
1. [output] → [initiative if applicable]
2. [output]
3. [output]

Strategic dependency: [what / from whom / when]
Decision threshold: [the decision that can't wait]
```

---

## Prioritization Triage

```
I have [N] competing items. Apply the prioritization hierarchy and give me a ranked top 3:

Items:
[list everything]

Hierarchy:
1. Commitment — what was promised to someone?
2. Leverage — what unblocks the most downstream work?
3. Reversibility — what, if not done today, forecloses options?

Output:
Top 3 (ranked): [with the hierarchy reason for each position]
Deferred (and why): [items below the cut with one-line rationale]
```

## Context Load Decision

```
I'm about to work on: [task or session]

Which 1-3 files or context items should I load?
Ceiling: 3. If more than 3 seem necessary, flag that the task scope may be too large.

Available context:
- execution/active-initiatives.md (initiative status)
- strategy/active-bets.md (active bets)
- knowledge/<domain>/ (domain knowledge)
- decision-frameworks/decisions-log.md (recent decisions)
- [any specific file I name]

Recommend: [file 1], [file 2 if needed], [file 3 if needed]
Reason for each: [one line]
If scope is too large: flag it and propose a scoped version of the task.
```
