## Skill Files

```
.claude/commands/         ← Executable commands (Claude Code loads these)
skills/README.md          ← This file — canonical registry (must match commands/ count)
```

---

## Skill Ownership Rules

1. Every skill in `.claude/commands/` must have a row in this table.
2. When a skill is invoked, update "Last Invoked" date.
3. New skills require: row in this table + test before marking Active.
4. New skill beyond count 120 requires naming and retiring a current skill first.

## Skill Design Reference

Full design spec: `architecture/SKILL-SYSTEM.md`

Skill file format:
```yaml
---
name: skill-name
description: >-  (what it does, trigger conditions, what it does NOT cover)
version: "X.Y"
changed: "YYYY-MM-DD — what changed"
output: [inline | file path]
---
```
