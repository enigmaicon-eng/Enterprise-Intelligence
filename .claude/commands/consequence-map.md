# /consequence-map

Map the downstream effects of a decision before committing. The goal is not to find reasons to avoid deciding — it is to expose the irreversible commitments hidden inside what looks like a simple choice.

## When to Use

- Before committing to any decision with low or irreversible reversibility
- When a decision feels right but you suspect there are hidden downstream effects
- As step 3 in the full pre-decision sequence: `/decision-recall` → `/pre-decide` → `/consequence-map` → `/decide`

**Do NOT use for:**
- Stress-testing the reasoning behind a claim → `/think`
- Generating or comparing options → `/pre-decide`
- Logging the decision → `/decide`
- Analysis paralysis — run this once per major choice, not recursively on every consequence

## Inputs Required

- The choice to map (inline): one option or decision direction

If multiple options were generated by `/pre-decide`: map them one at a time, not simultaneously.

## Workflow

**Step 1 — Parse the choice.**

Identify: what exactly is being decided? Name it as a specific action, not a direction. "We will adopt technology X" not "we will modernize."

**Step 2 — Map first-order consequences.**

Identify 3-5 direct, immediate consequences of making this choice. These are what happens as a direct result — not hypothetical downstream effects.

For each: assess reversibility:
- **Reversible** — can be undone within a sprint or similar short window
- **Manageable** — can be corrected with meaningful effort (weeks/months)
- **Irreversible** — cannot be undone without major cost or starting over
- **Unknown** — insufficient information to assess

**Step 3 — Map second-order effects.**

For each first-order consequence, identify 2-3 downstream effects that result from it. Focus on the ones that branch from the most consequential first-order outcomes.

Apply the same reversibility classification.

**Step 4 — Identify the highest-risk chain.**

The highest-risk chain is the path from the choice through the sequence of consequences where irreversibility compounds — where each step forecloses more options than the last.

If all chains are manageable: state that explicitly. Do not manufacture risk.

**Step 5 — Identify the key uncertainty.**

What single piece of information would most change the consequence map? Name it specifically — the one unknown that would most shift the reversibility assessment.

**Step 6 — Surface the consequence map.**

```
Consequence Map — [choice, one sentence]
══════════════════════════════════════════

If you commit to: [choice]

First-Order Consequences
  → [consequence 1]                          [reversible]
  → [consequence 2]                          [manageable]
  → [consequence 3]                          [irreversible]
  → [consequence 4]                          [unknown]

Second-Order Effects

  From [consequence 1]:
    → [effect A]                             [reversible]
    → [effect B]                             [manageable]

  From [consequence 2]:
    → [effect C]                             [manageable]
    → [effect D]                             [irreversible]

  From [consequence 3]:
    → [effect E]                             [irreversible]
    → [effect F]                             [irreversible]

Highest-Risk Chain
  [choice] → [consequence 3] → [effect E] → [effect F]
  Three irreversible steps. Once started, this path cannot be unwound cheaply.
  [— or "No high-risk chains detected. All consequences are manageable or reversible."]

Key Uncertainty
  [The one thing you'd need to know to reclassify the highest-risk consequence.]
  [E.g. "Whether [X] is true determines if consequence 3 is actually irreversible."]

Before Deciding
  [What to verify or observe before committing — specific, not generic]
```

**Step 7 — Offer /decide.**

```
Ready to commit?
  /decide  — log the decision with this consequence map as context
  /pre-decide [context]  — revisit options if this map changes the calculus
```

## Quality Gate

- [ ] Choice parsed to a specific action — not a direction or aspiration
- [ ] First-order consequences are direct effects, not downstream speculation
- [ ] Second-order mapping focuses on the most consequential first-order branches
- [ ] Reversibility classification applied per-consequence — not as a single blanket assessment
- [ ] Highest-risk chain identified by irreversibility compounding, not by intuition
- [ ] "No high-risk chains" stated explicitly if that is genuinely the case — not omitted
- [ ] Key uncertainty is specific and named — not "there are many unknowns"
- [ ] Output does not function as a recommendation to avoid deciding
